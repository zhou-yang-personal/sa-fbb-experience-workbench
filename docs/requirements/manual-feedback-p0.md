# Manual Feedback P0｜手动反馈问题与需求

本文件用于承接用户在自动 Review 之外手动记录的问题、需求和验收反馈。自动 Review 任务必须把本文件中未勾选项作为 P0 最高优先级处理。

## 执行规则

1. 每次自动 Review 开始时，必须在读取 `AGENTS.md`、`AGENTS.common.md`、`AGENTS.project.md` 后继续读取本文件。
2. 未勾选 `[ ]` 项 = P0 优先级，必须优先于自动发现的问题、重构、UI 优化、普通代码质量改进。
3. 一次自动 Review 任务必须尽力处理完全部未勾选项；不允许只挑容易项。确实无法处理时，必须在该项下写明阻塞原因、已完成部分和下一步。
4. 任务结束前必须回写本文件：
   - 已修复项改为 `[x]`；
   - 在该项下追加修复方案、涉及文件、验证结果 / 未验证原因、commit / PR；
   - 未修复项保持 `[ ]`，并更新进展 / 阻塞原因。
5. 不得删除历史反馈；需要归档时移动到「已处理记录」。
6. 如本文件为空或无未勾选项，自动 Review 再按常规 P0 规则选择最高优先级代码落地项。
7. 待处理区不得放包含字面量 `- [ ]` 的示例代码块，避免自动扫描误把模板当成真实 P0；示例请使用 `□` 占位。

## 待处理手动反馈

> 新增反馈写在本节顶部。格式示例：
>
> `- □ P0-YYYYMMDD-001｜标题`
> `  - 来源 / 时间：`
> `  - 问题 / 需求：`
> `  - 期望结果：`
> `  - 处理记录：`
