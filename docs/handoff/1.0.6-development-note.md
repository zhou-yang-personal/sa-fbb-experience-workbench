# 1.0.6 Development Note

## Scope

This round continues the code-completeness push after 1.0.5. The focus is to reduce large-entry-file coupling, add mapped RAW import, add mapping validation, and add modular UI shell code.

## Added code

- `src-tauri/src/raw_import_v2.rs`: mapped RAW import implementation using `cfg_import_field_mapping` aliases.
- `src-tauri/src/import_commands_mapped.rs`: adapter command that routes RAW import to `raw_import_v2`.
- `src-tauri/src/mapping_validation_commands.rs`: validates CSV headers against configured mappings and writes `meta_mapping_validation_result`.
- `src-tauri/src/inspection_commands.rs`: query helpers for quality gate, ETL steps and final action mix.
- `database/migrations/004_observability_schema.sql`: import row errors, mapping validation and dataset profile tables.
- `src/features/workbench/WorkbenchApp.tsx`: modular frontend shell for import, mapping, ETL, dashboard and leads workflow.
- `src/features/workbench/MetricGrid.tsx`: reusable metric cards.
- `src/features/workbench/ResultTables.tsx`: reusable SA Lead and Final Lead result tables.
- `src/features/workbench/ExecutionLog.tsx`: reusable execution log.
- `src/features/dashboard/EChartMetricBar.tsx`: ECharts bar chart panel for metric visualization.
