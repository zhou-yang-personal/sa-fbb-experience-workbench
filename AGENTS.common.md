# AGENTS.common.md｜AI 协作开发通用检查清单

本文件是从公共治理仓 `zhou-yang-personal/ai-dev-governance-kit` 的 `AGENTS.base.md` 同步而来。

本文件只放跨项目通用规则，不放任何单一项目的业务口径、技术细节或版本规则。

## A0. 执行入口

- [ ] 已读取当前项目根目录 `AGENTS.md`。
- [ ] 已按 `AGENTS.md` 要求继续读取 `AGENTS.common.md` 和 `AGENTS.project.md`。
- [ ] 如果上述任一文件无法读取，已暂停并要求用户提供文件内容，未直接设计或改代码。
- [ ] 已确认任务类型：需求设计 / 代码修改 / UI 调整 / PR 验收 / 文档治理 / 构建验证 / 其他。
- [ ] 已确认是否允许直接修改仓库；未确认时只输出方案，不写入。
- [ ] 涉及代码、配置、版本、数据库、构建或 PR 的变更前，已先输出设计方案。
- [ ] 用户明确确认后，才开始写入仓库。

## A4.1 Manual Feedback P0 文件协议

- [ ] 自动 review、hour-review、P0 review 等自动代码修改任务，开始后必须检查固定路径 `docs/requirements/manual-feedback-p0.md`。
- [ ] 如果 `docs/requirements/manual-feedback-p0.md` 存在，必须读取；如果不存在，记录“未发现”，然后按常规 P0 规则继续，不得报错中断。
- [ ] 如果文件存在但没有未勾选 `[ ]` 项，记录“无待处理手动反馈”，然后按常规 P0 规则继续。
- [ ] 如果文件存在且有未勾选 `[ ]` 项，未勾选项就是本轮 P0 最高优先级，必须优先于自动发现的问题、重构、UI 优化和普通代码质量改进。
- [ ] 自动 review 任务必须一次性尽力处理完该文件中的全部未勾选项；不得只挑容易项。
- [ ] 如果无法一次处理完某项，必须在交付报告中逐项说明阻塞原因、已完成部分和下一步。
- [ ] 自动 review 任务结束前必须回写 `docs/requirements/manual-feedback-p0.md`：已修复项改为 `[x]`，并简要记录修复方案、涉及文件、验证状态、commit / PR；未修复项保持 `[ ]` 并更新进展或阻塞原因。
- [ ] 每日公共治理同步任务只允许读取和分析 `docs/requirements/manual-feedback-p0.md`，不得自动覆盖或改写该文件。
